서베이 폼으로 일기 서비스를 만들고 싶어. 프론트는 자바스크립트로 동작하는 웹페이지고, UI는 vue.js를 이용해서 만들고 싶어. 모바일 UI로 웹에서도 볼 수 있게 할거야. DB는 스프레드시트에서 API를 만들어서 정보를 받아주고 싶어.

설문조사 폼은 아래와 같이 정의할께. 서비스하면서 추가 될 수 있으니 고려해서 구조를 만들어줘.
1. 간단한 주관식 형태
2. 객관식 점수 선택: 1~n점까지 점수가 있어서 하나를 클릭하는 형태
- 1점과 10점에 help text가 들어가 예를들면 1=아주 약함,10=아주강함
3.  yes. no를 선택하는 객관식 형태.
- yes, no는 대답에 따라 분기가 나뉘어 질 수 있어. 각 대답에 따라 몇 번 문항으로 갈건지 추가 셋팅이 가능해. 만약 없다면 그냥 다음 문제로 넘어가

예를들어) 질문이 다음과 같을 떄
1. 운동을 했어? Yes (to 2) / No (to 3)
2. 운동 강도는 얼마나 했어? (객관식 점수 선택: 5점, help text 1=가벼운운동,5=힘든운동)
3. 밥은 먹었어? Yes / No
4. 오늘의 기분을 간단하게 써줘.

이러면 1번에서 yes를 선택하면 2번으로 가고, no를 선택하면 3번으로 가. 3번 문제는 무엇을 선택하든 4번 문제로 가. 2번 질문은 간단한 주관식으로 input box가 짧게 있어.

상단화면의 1/2은 달력이 나오고 오늘 날자가 선택되어 있어. 딜력에서 날자를 바꾸면 해당 설문지아 아래 나오는 형태야.

주요기능은
일기 확인, 일기 쓰기, 일기 수정이 있어. 모든 일기는 간단한 설문조사 폼으로 되어 있어.

일기 확인은 스프레드 시트에 아이디와 함께 저장된 날자의 데이터를 질문의 답이 입력된 형태로 되어 있어.
일기 쓰기는 날자에 일기가 없으면 지정된 질문지의 설문을 받는 형태야. 설문은 질문지 리스트로 언제나 수정할 수 있어.
일기 수정은 일기를 확인한 상태에서 수정하면 저장 버튼이 오른쪽 위에 나왔으면 해. 만약 수정했는데, 페이지를 벗어나려고 하면 수정된게 저장할 거냐고 물어봐줘.

위 내용을 바탕으로 폴도 구조와 파일명을 테이블로 정리하고 각 파일에 맞는 객체지향 프로그래밍 코드를 작성해줘. 질문지 리스트는 세가지 설문조사 form으로 언제나 쓸 수 있게 따로 빼줘.
