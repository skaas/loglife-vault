# Loglife Compile Contract

이 저장소는 삶의 기록을 단순히 보관하는 폴더가 아니라, 원본 기록을 자기지식과 공개 프로필로 컴파일하는 개인 지식 시스템이다.

- `Inbox`는 원본 입력이다.
- `Daily`는 날짜 맥락을 정리하는 중간 산출물이다.
- `Wiki`는 시간이 지나도 남는 자기지식의 목적지다.
- `site`는 외부에 보여줄 공개 산출물이다.
- `Meta`는 컴파일 규칙과 품질 검사를 담당한다.

## 핵심 원칙

- 이 저장소의 목표는 기록을 많이 쌓는 것이 아니라, 시간이 갈수록 `나에 대한 더 정확한 맵`을 만드는 것이다.
- `Inbox` 원본은 가능한 한 수정하지 않는다.
- 원본 파일명은 Windows에서도 안전해야 한다. `?`, `*`, `:`, `|` 같은 금지 문자를 쓰지 않는다.
- `Daily`는 하루의 흐름을 정리하는 중간 산출물이다.
- `Wiki`는 시간이 지나도 남는 지식을 누적하는 목적지다.
- `site`는 `Wiki` 전체 공개가 아니라, 외부에 보여도 되는 내용만 다시 컴파일한 결과물이다.
- 모든 해석은 가능하면 원본 링크를 근거로 남긴다.
- 사실, 자기평가, 감정, 추측을 구분해서 적는다.

## 비목표

- 모든 기록을 전부 `Wiki`로 올리는 것
- 모든 해석을 사실처럼 확정하는 것
- 내부 `Wiki`를 그대로 공개하는 것

## 컴파일 파이프라인

1. `Inbox` 항목을 읽는다.
2. 원본에서 사실, 자기서술, 감정, 질문, 링크를 분리한다.
3. 날짜 맥락은 `Daily`에 반영한다.
4. 지속 가치가 있는 내용만 `Wiki`에 승격한다.
5. 기존 문서가 있으면 새 문서를 만들기보다 기존 문서를 갱신한다.

## 라우팅 규칙

- 비교적 안정적인 자기서술:
  `Wiki/Self/Profile.md`
- 반복되는 내적 충돌이나 딜레마:
  `Wiki/Self/Tensions.md`
- 아직 닫히지 않은 자기 질문:
  `Wiki/Self/Open Questions.md`
- 특정 주제의 반복 패턴:
  `Wiki/Themes/*.md`
- 사람과 관계의 누적 정보:
  `Wiki/People/*.md`
- 특정 사건으로 묶이는 기록:
  `Wiki/Events/*.md`
- 지속적인 일과 목표:
  `Wiki/Projects/*.md`
- 시간순 회고와 전환점:
  `Wiki/Timeline/*.md`

## 링크 규칙

- `Inbox/Text` 또는 `Inbox/Telegram` 안에 링크가 있으면 원문 URL을 보존한다.
- 링크를 읽을 수 있으면 핵심 내용을 짧게 요약한다.
- 링크 요약은 원본을 대체하지 않는다.
- 링크에서 얻은 내용은 관련 `Daily` 또는 `Wiki` 문서의 근거로만 사용한다.

## 자기지식 승격 기준

아래 중 하나에 해당하면 `Wiki`로 승격한다.

- 같은 주제가 반복된다.
- 자기 이해에 도움이 되는 문장이다.
- 중요한 가치관, 욕구, 습관, 한계가 드러난다.
- 나중에 다시 보고 싶은 질문이 생긴다.
- 특정 사람이나 사건과 반복적으로 연결된다.

## 문장 규칙

- 단정 대신 `기록상`, `스스로는`, `이 시점에는` 같은 표현을 우선한다.
- 해석은 짧고, 근거 링크는 분명하게 남긴다.
- 중복 문장이 생기면 더 넓은 상위 개념 문서로 합친다.
- 같은 의미의 문장이 다른 워딩으로 반복되면 `Meta/wording-map.yaml`을 먼저 확인한다.
- 승인된 정규 표현이 없으면 임의로 통일하지 말고 `Meta/wording-conflicts.md`에 open conflict를 추가한다.
- 단어 충돌은 후처리 메모가 아니라 `컴파일 warning`으로 본다.
- 커밋 전에는 `scripts/check-windows-paths.sh --staged`가 통과해야 한다.
